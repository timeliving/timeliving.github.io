Java中的字符串

##1.不可变的String对象

String对象是不可变，以下是String类的源码：

public final class String
implements java.io.Serializable, Comparable<String>, CharSequence
{
    /** The value is used for character storage. */
    private final char value[];

    /** The offset is the first index of the storage that is used. */
    private final int offset;

    /** The count is the number of characters in the String. */
    private final int count;

    /** Cache the hash code for the string */
    private int hash; // Default to 0

    /** use serialVersionUID from JDK 1.0.2 for interoperability */
    private static final long serialVersionUID = -6849794470754667710L;

    ........
}

从以上可看出：

  1. String类是final类，也就意味着String不能被继承
  2. 从上面可看出String类是使用char数组保存字符串的

执行以下代码可发现String类是不可改变的

public class Immutable {
    public static String upcase(String s){
        return s.toUpperCase();
    }

    public static void main(String[] args){
        String q="howdy";
        System.out.print(q);
        String qq = upcase(q);
        System.out.print(qq);
        System.out.print(q);
    }
}

以上代码输出结果是howdy HOWDY howdy由此可得出当修改一个String对象时，实际上是生成一个新的String对象并返回一个新的对象的引用。

因此String对象一旦被创建就是固定不变的，对String对象的任何改变都不影响到原对象，任何change操作都会生成新的对象

以下是String类的一些方法实现，从中也可看出String对象是不可变的

public String substring(int beginIndex, int endIndex) {
    if (beginIndex < 0) {
        throw new StringIndexOutOfBoundsException(beginIndex);
    }
    if (endIndex > count) {
        throw new StringIndexOutOfBoundsException(endIndex);
    }
    if (beginIndex > endIndex) {
        throw new StringIndexOutOfBoundsException(endIndex - beginIndex);
    }
    return ((beginIndex == 0) && (endIndex == count)) ? this :
        new String(offset + beginIndex, endIndex - beginIndex, value);
}

public String concat(String str) {
    int otherLen = str.length();
    if (otherLen == 0) {
        return this;
    }
    char buf[] = new char[count + otherLen];
    getChars(0, count, buf, 0);
    str.getChars(0, otherLen, buf, count);
    return new String(0, count + otherLen, buf);
}

public String replace(char oldChar, char newChar) {
    if (oldChar != newChar) {
        int len = count;
        int i = -1;
        char[] val = value; /* avoid getfield opcode */
        int off = offset;   /* avoid getfield opcode */

        while (++i < len) {
        if (val[off + i] == oldChar) {
            break;
        }
        }
        if (i < len) {
        char buf[] = new char[len];
        for (int j = 0 ; j < i ; j++) {
            buf[j] = val[off+j];
        }
        while (i < len) {
            char c = val[off + i];
            buf[i] = (c == oldChar) ? newChar : c;

        }
        return new String(0, len, buf);

    }

}

##2.字符串常量池

JVM为了提高性能和减少内存开销，在实例化字符串的时候进行了一些优化：使用字符串常量池，每当我们创建字符串常量时，JVM会首先检查字符串常量池，如果该字符串已经存在常量池中，那么就直接返回常量池中的实例引用。如果字符串不存在常量池中，就会实例化该字符串，并且将其放到常量池中。

Java中的常量池，实际上分为两种形态：静态常量池和运行时常量池。

所谓静态常量池，即*.class文件中的常量池，class文件中的常量池不仅仅包含字符串(数字)字面量，还包含类、方法的信息，占用class文件绝大部分空间。

而运行时常量池，则是jvm虚拟机在完成类装载操作后，将class文件中的常量池载入到内存中，并保存在方法区中，我们常说的常量池，就是指方法区中的运行时常量池。

例：字符串常量池的使用

String s0 = "qwe"; 
String s1 = "qwe"; 
System.out.println(s0==s1); //true  可以看出s0和s1是指向同一个对象的。

s0、s1都是指向JVM字符串常量池中的”qwe”对象

而当使用new关键字创建字符串时就完全不同了，例：

String s2 = new String("qwe");

首先，new关键字一定会产生一个对象qwe，且存储在堆中，所以在执行以下代码时结果就不相同了

System.out.println(s1==s2); //false  可以看出s1和s2是指向不同对象的。

以上s0、s1和s2的关系可见下图

总结：虽然s0、s1和s2的值相同，但是s2是指向时创建在堆中的对象。

部分内容出处——翡青

##Class类的反射机制

  • 指的是可以于运行时加载,探知和使用编译期间完全未知的类.
  • 程序在运行状态中, 可以动态加载一个只有名称的类, 对于任意一个已经加载的类,都能够知道这个类的所有属性和方法; 对于任意一个对象,都能调用他的任意一个方法和属性;
  • 加载完类之后, 在堆内存中会产生一个Class类型的对象(一个类只有一个Class对象), 这个对象包含了完整的类的结构信息,而且这个Class对象就像一面镜子,透过这个镜子看到类的结构,所以被称之为:反射。
  • 每个类被加载进入内存之后,系统就会为该类生成一个对应的java.lang.Class对象,通过该Class对象就可以访问到JVM中的这个类.

##Class对象的获取

  • 对象的getClass()方法;
  • 类的.class(最安全/性能最好)属性;
  • 运用Class.forName(String className)动态加载类,className需要是类的全限定名(最常用).

Use Markdown

1.标题设置

在Markdown中设置标题，有两种方式：

第一种Setex。
其中<h1>和<h2>分别可用=和-在文字下方添加代替。

第二种atx。
是在文字开头加上#，通过#数量表示几级标题。(一共只有1~6级标题，1级标题字体最大)

2.块引用

在文字开头添加使用>来表示块引用。(当>和文字之间添加五个blank space是，块引用的文字会有变化。)

3.强调

  1. 将需要设置为斜体的文字用*或_包裹起来。
  2. (其中`和**`包裹表示粗体)__
  3. 在空白行下方添加三条“-”横线表示下划线。

4.无序、有序列表

无序（项目符号）列表使用星号，加号和连字符（*， +，和-）作为列表标记，这三个标记是可互换的。(便于查看推荐一个页面只是用一种标记。)

有序（编号）列表使用常规数字，后跟字符句号加空格作为列表标记

5.链接

Markdown支持两种创建链接的样式：内联和引用。

第一种内联样式

在链接文本之后立即使用括号，例如This is an [example link](http://example.com/).
或者，也可以在括号中包含标题属性：This is an [example link](http://example.com/ "With a Title").

第二种引用样式

参考内联样式可以通过在文档中其他位置定义的名称来引用所需的链接，例如：

I get 10 times more traffic from [Google][1] than from
[Yahoo][2] or [MSN][3].

[1]: http://google.com/        "Google"
[2]: http://search.yahoo.com/  "Yahoo Search"
[3]: http://search.msn.com/    "MSN Search"

其中title属性是可选的，链接名称可包含字母、数字和空格，但不区分大小写

6.图像

图像语法非常接近链接语法。同样分为内联和引用两种。

第一种内联样式
![alt text](/path/to/img.jpg "Title")

第二种引用样式

![alt text][id]

[id]: /path/to/img.jpg "Title"

7.代码(HTML中的Code)

第一种：使用反撇号(`)包围代码即可，支持换行，((`)不是单引号而是左上角的ESC下面~中的(`)) 例如：
I strongly recommend against using any <blink> tags.

I wish SmartyPants used named entities like `&mdash;`
instead of decimal-encoded entites like `&#8212;`.

第二种：指定整个代码块，将代码块的每一行缩进4个空格或1个Tab。其中&，<，和>字符将被自动转义。例如：

If you want your page to validate under XHTML 1.0 Strict,
you've got to put paragraph tags in your blockquotes:

    <blockquote>
        <p>For example.</p>
    </blockquote>

以上内容根据创始人 John Gruber 的 Markdown 语法说明进行整理。

Markdown官方网站：http://daringfireball.net/projects/markdown/

用JAVA获取文件，听似简单，但对于很多像我这样的新人来说，还是掌握颇浅，用起来感觉颇深，大常最经常用的，就是用JAVA的File类，如要取得c:/test.txt文件，就会这样用File file = new File("c:/test.txt");这样用有什么问题，相信大家都知道，就是路径硬编码，对于JAVA精神来说，应用应该一次成型，到处可用，并且从现实应用来讲，最终生成的应用也会部署到Windows外的操作系统中，对于linux来说，在应用中用了c:/这样的字样，就是失败，所以，我们应该尽量避免使用硬编码，即直接使用绝对路径。

在Servlet应用中，有一个getRealPath(String str)的方法，这个方法尽管也可以动态地获得文件的路径，不秘直接手写绝对路径，但这也是一个不被建议使用的方法，那么，我们有什么方法可以更好地获得文件呢?

那就是Class.getResource()与Class.getResourceAsStream()方法，但很多人还是不太懂它的用法，因为很多人（比如不久前的我）都不知道应该传怎么样的参数给它，当然，有些人己经用得如火纯青，这些人是不需要照顾的，在此仅给不会或者还不是很熟的人解释一点点。
